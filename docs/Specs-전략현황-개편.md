## 목표 (Objective)

`⚙️ 전략현황` 탭을 개편하여, 상단에 전략별 실제 성과 테이블을 배치하고 전략 설명 카드를 좌우 스와이프 형태로 전환한다. 탭 수를 4개로 유지하면서 성과 분석 정보를 제공한다.

---

## 핵심 요구사항 (Requirements)

- `Must` 전략별 성과 테이블을 탭 최상단에 표시 (거래수 / 승률 / 평균수익률 / 총손익)
- `Must` 전략 설명 카드를 좌우 스와이프 방식으로 변경 (scroll-snap 기반)
- `Must` 매도 이력 없는 전략은 성과 테이블에서 "-" 표시
- `Should` 데스크탑에서는 카드 그리드 유지, 모바일에서만 스와이프
- `Nice` 성과 테이블에서 수익/손실 컬러 구분 (빨강/파랑)

## 유저 시나리오 (User Story)

**As a** 포트폴리오 운용자,
**I want to** 전략별 실제 성과를 한눈에 확인하고 전략 설명을 스와이프로 탐색하고 싶다,
**So that** 어떤 전략이 잘 작동하는지 파악하고 운용 방향을 조정할 수 있다.

## 검수 기준 (Acceptance Criteria)

- [ ] **Given** 전략현황 탭 진입
      **When** `trades` 테이블에 매도 이력 존재
      **Then** 전략별 거래수 / 승률 / 평균수익률 / 총손익이 테이블로 표시된다

- [ ] **Given** 전략현황 탭 진입
      **When** 매도 이력 없음
      **Then** 성과 테이블의 수치 항목이 "-"로 표시된다

- [ ] **Given** 모바일 환경 (width ≤ 768px)
      **When** 전략 카드 영역에서 좌우 스와이프
      **Then** 카드가 1개씩 스냅되며 전환된다

- [ ] **Given** 데스크탑 환경 (width > 768px)
      **When** 전략현황 탭 진입
      **Then** 전략 카드가 기존 그리드 레이아웃으로 표시된다

- [ ] **Given** 총손익 양수
      **When** 성과 테이블 렌더링
      **Then** 빨간색으로 표시된다 (수익)

- [ ] **Given** 총손익 음수
      **When** 성과 테이블 렌더링
      **Then** 파란색으로 표시된다 (손실)
